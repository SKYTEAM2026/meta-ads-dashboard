# 메타 대시보드 — 소재 ON/OFF · 예산 수정 작업 노트

작성일: 2026-06-28

---

## 1. 이번 작업 요약

메타 대시보드(`index.html`)에 **광고를 조회만 하던 것**에서 **직접 제어**할 수 있는 기능을 추가했다.

| 기능 | 위치 | 동작 |
|------|------|------|
| 소재 ON/OFF | 소재별 탭, 광고세트 확장 표 | 상태 뱃지 클릭 → 활성↔일시중지 토글 |
| 광고세트 ON/OFF | 광고세트 표 | 상태 뱃지 클릭 → 토글 |
| 캠페인 ON/OFF | 캠페인 성과 표 | 상태 뱃지 클릭 → 토글 |
| 예산 수정 (CBO) | 캠페인 표 | 예산 옆 연필 버튼 → 금액 입력 |
| 예산 수정 (ABO) | 광고세트 표 | 예산 옆 연필 버튼 → 금액 입력 |

- 상태 변경/예산 수정 모두 **확인창** 또는 **입력창**을 거쳐 실행 → 실수로 눌러도 안전.
- CBO 캠페인은 광고세트 표에 `캠페인 예산(CBO)` 표시, ABO 캠페인은 캠페인 표에 `광고세트 예산` 표시(해당 레벨에서 수정).

## 2. 추가/변경된 코드 (index.html)

- **`apiPost(path, params)`** — 쓰기용 POST 헬퍼 (기존 `api()`는 조회용 GET).
- **`toggleEntityStatus(ev, id, btn)`** — 광고/광고세트/캠페인 공용 상태 토글.
- **`editBudget(ev, id, reload)`** — 캠페인/광고세트 공용 예산 수정.
  - `reload === 'campaign'` → 로컬 state 갱신 후 `renderCampaignTable` 재렌더
  - `reload === 'adset'` → `adsetTabLoaded=false` 후 `loadAdsetTab` 재호출
- 상태 뱃지를 `.status-toggle` 클래스 + `onclick`으로 변경 (행 클릭과 충돌 방지 위해 `stopPropagation`).
- 예산 셀에 `.budget-edit-btn` 연필 버튼 추가.
- CSS: `.status-toggle`, `.budget-edit-btn` 추가.

### 통화/예산 단위
- 계정 통화가 **KRW** → Meta 예산 minor 단위가 **원**이라 입력값(원)을 그대로 전송.
  (USD 등은 minor=센트라 환산 필요하지만 본 계정은 KRW.)

## 3. ⚠️ 토큰 권한 (필수)

- **조회**: `ads_read`
- **소재 ON/OFF · 예산 수정**: `ads_management` 반드시 필요.
- 권한 없는 토큰으로 변경 시도 시 → 실패 alert 표시(안전).
- 토큰은 대시보드 우측 상단 **토큰 설정**에서 교체. 브라우저 로컬스토리지에만 저장.

---

## 4. 토큰 갱신 가이드

### (A) 단기 토큰 — Graph API Explorer
1. https://developers.facebook.com/tools/explorer/
2. 권한에 `ads_read` + `ads_management` 추가
3. `Generate Access Token` → 권한 허용 → 토큰 복사 → 대시보드에 저장
- **수명: 약 1~2시간** (만료되면 재발급해서 교체)

### (B) 60일 장수 토큰 — 시크릿으로 교환 (권장)
브라우저 주소창에 입력 (대문자 3곳 치환):
```
https://graph.facebook.com/v21.0/oauth/access_token?grant_type=fb_exchange_token&client_id=앱ID&client_secret=앱시크릿&fb_exchange_token=단기토큰
```
- 응답 JSON의 `access_token`이 60일 토큰 (`expires_in: 5184000` ≈ 60일)
- **앱 ID/시크릿 위치**: developers.facebook.com/apps → 앱 → 앱 설정 → 기본 설정 (시크릿은 `표시` 클릭)
- 시크릿은 **한 번만** 확보하면, 이후 만료 시 재교환만 하면 됨(로그인 불필요).

### (C) 무기한 토큰 — 시스템 사용자 (최선, 로그인 필요)
- 비즈니스 관리자 → 시스템 사용자 토큰 생성 → 만료 없음 설정 가능.

### 현재 상태 (2026-06-28)
- 단기 토큰으로 동작 확인 완료(ON/OFF·예산수정 정상).
- **60일 교환 보류**: Meta 비밀번호 분실로 앱 시크릿 확인 불가.
- → 비번 복구 후 **앱ID + 시크릿 + 단기토큰** 확보하면 60일/무기한 토큰으로 전환 가능.

### 교환 오류 대처
- `Error validating client secret` → 시크릿 오타/재복사
- `Invalid OAuth access token` → 단기 토큰 만료, Explorer에서 재발급
- `expires_in`이 짧거나 없음 → 단기 토큰이 아닌 다른 토큰 사용함

## 5. 배포 주의 (auto-push 훅)

- 파일 수정 시 자동 commit→push 훅이 있으나, **commit만 되고 push 실패가 누적**된 적 있음.
  (원격의 카페24 매출 자동 업데이트 커밋과 분기되면 push 거부됨.)
- 대시보드에 수정이 안 보이면:
  ```
  git fetch origin
  git rebase origin/main
  git push origin HEAD
  ```
- 푸시 후 GitHub Pages 반영까지 ~1분, 브라우저는 **Ctrl+Shift+R**(강력 새로고침)로 캐시 우회.

## 6. 사용법 (요약)

1. 대시보드에서 캠페인 클릭 → 광고세트/소재별 탭 진입
2. **끄기/켜기**: 상태 뱃지(활성/일시중지) 클릭 → 확인 → 즉시 반영
3. **예산 변경**: 예산 옆 연필 아이콘 클릭 → 원 단위 금액 입력 → 확인 → 반영
4. 토큰 만료로 오류 나면 → 새 토큰 발급해서 토큰 설정에 교체
